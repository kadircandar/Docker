Docker Volume Nedir?

Kalıcı veri saklamanın en kolay yolu

Docker Volume, Docker container'larının içinde oluşturduğu verilerin container silinse bile kaybolmamasını sağlayan kalıcı depolama birimidir.

Sorun Nedir?

Normalde bir container'ı silerseniz, içindeki tüm dosyalar da silinir:

$ docker run -it ubuntu bash
# içinde dosya oluşturduk...
$ docker rm    # → Tüm veriler GİTTİ!

Çözüm: Docker Volume

Volume kullanarak verileri container'dan bağımsız hale getiririz.

En Kolay Kullanım (Tavsiye Edilen)

# Otomatik olarak volume oluşturur ve bağlar
docker run -d \
  --name my-web-app \
  -v mydata:/app/data \     # ← burası önemli
  -p 8080:80 \
  nginx

Diğer Volume Türleri

  • Named Volume (önerilen): -v veritabani:/var/lib/mysql
  • Anonymous Volume: -v /app/logs (isim verilmez)
  • Bind Mount: -v /host/klasor:/container/klasor
İpucu: Günlük hayatta %90 named volume yeterlidir.
-v isim:/container/içindeki/yol şeklinde kullanılır.

Pratik Örnekler

# MySQL ile kalıcı veritabanı
docker run -d --name mysql-db \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -v mysql-data:/var/lib/mysql \
  mysql:latest

# WordPress sitesi (veriler kaybolmaz)
docker run -d --name wp-site \
  -v wp-content:/var/www/html/wp-content \
  -p 8000:80 \
  wordpress

Volume Komutları

# Tüm volumeları listele
docker volume ls

# Volume detayını gör
docker volume inspect mysql-data

# Kullanılmayan volumeları temizle
docker volume prune

Sonuç: Docker Volume kullanmazsanız verileriniz container ile birlikte ölür. Her zaman önemli veriler için volume kullanın!
